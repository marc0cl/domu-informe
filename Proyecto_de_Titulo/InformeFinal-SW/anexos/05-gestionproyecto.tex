\subsection{Análisis de Desviaciones}

Al comparar la carta Gantt del anteproyecto (Figura~\ref{fig:gantt-anteproyecto}) con la ejecución real (Figura~\ref{fig:gantt-proyecto}), se identifican las siguientes desviaciones principales:

\begin{itemize}
    \item \textbf{Fase de diseño de base de datos:} Se extendió aproximadamente una semana más de lo planificado debido a la complejidad del modelo financiero (gastos comunes, períodos contables, tipos de cargo). La decisión de modelar 35~entidades en 9~dominios, en lugar de las ~20 estimadas inicialmente, impactó esta fase.

    \item \textbf{Fase de implementación del backend:} La construcción de los módulos de finanzas y proveedores demandó más iteraciones de las previstas, particularmente la generación de documentos PDF y la integración con Box para almacenamiento. Se compensó parcialmente con una implementación más rápida del frontend.

    \item \textbf{Módulos postergados a v2.0:} Los casos de uso CU\_06 (conciliación bancaria), CU\_08 (turnos de personal), CU\_11 (mantenimiento preventivo), CU\_12 (estacionamientos) y CU\_18 (bitácora de ascensores) fueron postergados durante la ejecución para asegurar la calidad de los módulos entregados. Esta decisión se tomó en la semana~8 del desarrollo.

    \item \textbf{Fecha de cierre:} A pesar de las desviaciones puntuales, la fecha de cierre global del proyecto se mantuvo dentro del margen planificado, gracias a la re-priorización de funcionalidades y la eficiencia ganada por la adopción de TanStack React Query en el frontend.
\end{itemize}

\subsection{Puntos de Casos de Uso Ajustados (UCP)}

\[
\text{UCP} = \text{UUCP} \times \text{TCF} \times \text{EF} = 278 \times 1.055 \times 0.89 = \mathbf{260.9}
\]

\subsection{Nivel de Esfuerzo (LOE)}

Para determinar las horas por punto de caso de uso, se evalúan los factores ambientales:
\begin{itemize}
    \item Factores E1--E6 con valor $< 3$: E2 (valor~2) $\rightarrow$ \textbf{1 factor}.
    \item Factores E7--E8 con valor $> 3$: ninguno $\rightarrow$ \textbf{0 factores}.
    \item Total: \textbf{1 factor} $\leq 2$ $\Rightarrow$ LOE = \textbf{20 hrs/UCP}.
\end{itemize}

\subsection{Esfuerzo Estimado}

\[
\text{Esfuerzo} = \text{UCP} \times \text{LOE} = 260.9 \times 20 = \mathbf{5{,}218 \text{ horas-persona}}
\]

\subsection{Análisis comparativo}

La estimación UCP arroja un valor teórico de 5{,}218~horas-persona, considerablemente superior a las 350~horas reales invertidas. Esta divergencia se explica por varios factores:

\begin{itemize}
    \item \textbf{Factor LOE estándar:} El método UCP utiliza un LOE de 20~hrs/UCP como referencia para equipos profesionales a tiempo completo. En el contexto de un proyecto de título con dos desarrolladores que reutilizaron componentes y librerías existentes, la productividad por punto fue significativamente mayor.

    \item \textbf{Reutilización de tecnología:} El uso de frameworks maduros como Javalin, TanStack React Query y Flyway redujo el esfuerzo de implementación respecto a un desarrollo desde cero.

    \item \textbf{CUs diferidos:} Cinco de los 27 CUs (los más complejos: CU\_06, CU\_08, CU\_11, CU\_12, CU\_18) fueron diferidos a v2.0, reduciendo el esfuerzo real sin modificar la estimación teórica.

    \item \textbf{Factor de ajuste empírico:} Aplicando un factor de ajuste de $\approx 0.067$ (350/5{,}218), se obtiene un LOE efectivo de $\approx 1.34$~hrs/UCP, coherente con proyectos académicos de alta productividad reportados en la literatura.
\end{itemize}
